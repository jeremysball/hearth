# Writing Clearly and Concisely

## Overview

William Strunk Jr.'s *The Elements of Style* (1918) teaches you to write clearly and cut ruthlessly. `elements-of-style.md` is a compact, ~700-token distillation — cheap enough to read every time, not just for long-form writing.

## When to Use This Skill

Use this skill for any prose a human will read, no matter how short:

- Documentation, README files, technical explanations
- Commit messages, PR titles and descriptions, issue comments
- Error messages, UI copy, help text, code comments
- Reports, summaries, explanations, chat replies
- A single sentence you're about to send

**Default to using it.** If you're writing words for a human, read `elements-of-style.md` first — it's small enough that there's no real cost to checking.

## All Rules

### Composition (the rules that matter most)
- Active voice — including passive-perfect and actor-dropped-passive disguises
- Positive form — say what is, not what isn't
- Specific and concrete language — name the thing, the number, the actor
- Omit needless words — cut hedges, throat-clearing, stock closers, intensifiers
- Don't hedge facts with weak modals (*can/may/should* as tics)
- Collapse "so that X can Y" constructions
- Parallel structure across a series
- Keep related words together
- One topic per paragraph, topic sentence first
- Place emphatic words at the end of the sentence

### Grammar (rarely needed on LLM output — check only for human-drafted or rough text)
Possessives, series commas, parenthetical commas, comma splices, misplaced modifiers, tense consistency.

## Bottom Line

Writing for humans? Read `elements-of-style.md` and apply the rules — every time, not just for long documents.
