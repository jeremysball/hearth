## Task 2: Speed up bar enter animations and add spring easing

**Problem:** `animateGrow` uses 450ms duration with 35ms per-bar stagger and a flat `ease-out` curve. Two changes: (a) cut duration to 250ms and stagger to 20ms for snappiness; (b) add an optional `easing` param defaulting to a spring curve so scale animations overshoot and pop.

**Note:** This overrides the duration baked into ADR-0002. The tests in `adr0002-animations.test.js` check presence and completion within 900ms ‚Äî they survive this change.

**Design:** Spring cubic-bezier `cubic-bezier(0.34, 1.56, 0.64, 1)` overshoots `scaleY(1)` slightly, giving bars a satisfying pop. Animations that drive `strokeDasharray`/`opacity`/`strokeDashoffset` (sleep rings, growth polyline, growth area) keep `ease-out` to avoid visual glitches ‚Äî pass them `'ease-out'` explicitly.

- Modify: `js/app.js` (`enterTrends` stagger; `enterSleep`/`enterGrowth` pass explicit `ease-out`)
- Modify: `index.html` (version bump)
- Modify: `sw.js` (version bump)

**Interfaces:**
- Consumes: `animateGrow(el, keyframes, delayMs?, easing?)` ‚Äî adds optional 4th param
- Produces: same export, now spring by default


Replace it with:
```js
function hapticAudio(ms) {
  if (state().settings.sound === false) return;
  const ctx = getCtx(); if (!ctx) return;
  const dur = Math.max(0.004, ms / 1000);
  const buf = ctx.createBuffer(1, Math.ceil(ctx.sampleRate * dur), ctx.sampleRate);
  const data = buf.getChannelData(0);
  for (let i = 0; i < data.length; i++) data[i] = (Math.random() * 2 - 1) * (1 - i / data.length);
  const src = ctx.createBufferSource();
  const gain = ctx.createGain();
  src.buffer = buf;
  gain.gain.setValueAtTime(0.025, ctx.currentTime);
  src.connect(gain).connect(ctx.destination);
  src.start();
}

export function buzz(ms) {
  if (navigator.vibrate) {
    navigator.vibrate(ms);
  } else {
    hapticAudio(ms);
  }
}
```

`hapticAudio` produces a 4‚Äì15ms white noise burst at 0.025 gain ‚Äî barely audible, designed to feel like a click. The amplitude envelope tapers naturally because the noise data multiplied by `(1 - i/data.length)`.
